# A股股票及ETF均线监控需求文档

## 1. 项目概述

### 1.1 项目背景

本项目旨在构建一个自动化监控系统，用于实时监控 A股市场股票及ETF的价格走势。当股价上穿或下穿20日移动平均线（MA20）时，系统自动推送通知，帮助投资者及时把握交易信号。

### 1.2 项目目标

- 实现多只A股股票和ETF的自动化监控
- 每隔10分钟自动获取最新股价
- 计算并判断股价与20日均线的关系
- 当股价突破均线时发送即时通知
- 支持自定义监控列表

---

## 2. 功能需求

### 2.1 核心功能

| 序号 | 功能描述 | 优先级 |
|------|----------|--------|
| F1 | 监控列表管理 - 支持添加/删除/查看监控的股票和ETF代码 | P0 |
| F2 | 实时价格获取 - 自动获取A股实时行情数据 | P0 |
| F3 | MA20计算 - 基于历史数据计算20日移动平均线 | P0 |
| F4 | 穿越检测 - 检测股价上穿/下穿MA20的信号 | P0 |
| F5 | 定时任务 - 每10分钟自动执行监控检查 | P0 |
| F6 | 消息通知 - 通过Telegram发送突破信号通知 | P0 |
| F7 | 状态持久化 - 保存上一次的均线状态，避免重复通知 | P1 |

### 2.2 监控逻辑

```
每个监控周期（10分钟）执行以下逻辑：

1. 获取监控列表中的所有股票/ETF
2. 对每只标的：
   a. 获取最新收盘价
   b. 获取过去20个交易日的收盘价
   c. 计算MA20 = 过去20日收盘价的算术平均值
   d. 判断当前位置: 价格 > MA20 (多头) / 价格 < MA20 (空头)
   e. 与上次状态对比:
      - 从"below"变为"above" → 买入信号 🔔
      - 从"above"变为"below" → 卖出信号 🔔
   f. 发送通知（仅当状态发生变化时）
3. 更新并保存当前状态
```

### 2.3 数据要求

#### 2.3.1 股票代码格式

| 市场 | 代码示例 | 格式说明 |
|------|----------|----------|
| 上海证券交易所 | 600519、510300 | 以6开头，通常为6位数字 |
| 深圳证券交易所 | 000001、159915 | 以0或3开头，通常为6位数字 |
| 科创板 | 688001 | 以688开头 |

#### 2.3.2 监控标的类型

- **A股股票**: 如贵州茅台(600519)、宁德时代(300750)
- **ETF基金**: 如沪深300ETF(510300)、中证500ETF(510500)、创业板ETF(159915)

## 3. 非功能需求

### 3.1 性能要求

- 单次全量检查响应时间 < 30秒
- 单只股票价格获取 < 3秒
- 系统资源占用低，可长时间运行

### 3.2 可用性要求

- 支持7x24小时连续运行
- 网络异常时具备重试机制
- 状态文件损坏时自动重建

### 3.3 可配置性

- 监控间隔时间可配置
- 股票列表可通过配置文件修改
- 通知方式可扩展

---

## 4. 通知需求

### 4.1 通知触发条件

| 场景 | 通知内容 | 优先级 |
|------|----------|--------|
| 股价上穿MA20 | 🚀 {股票代码} 突破20日均线 - 现价: {price}, MA20: {ma20} | 高 |
| 股价下穿MA20 | 🔻 {股票代码} 跌破20日均线 - 现价: {price}, MA20: {ma20} | 高 |
| 程序启动 | ✅ 监控服务已启动 - 监控 {count} 只标的 | 低 |

### 4.2 通知渠道

- **主渠道**: Telegram Bot（推荐）
- **备选渠道**: 微信企业群机器人（预留扩展）

## 5. 配置文件结构

```yaml
# config.yaml
monitor:
  interval: 600  # 监控间隔，单位秒（默认10分钟）
  
stocks:
  - "600519"   # 贵州茅台
  - "000001"   # 平安银行
  - "300750"   # 宁德时代
  - "688001"   # 华兴源创

etfs:
  - "510300"   # 沪深300ETF
  - "510500"   # 中证500ETF
  - "159915"   # 创业板ETF
  - "513100"   # 纳指ETF

notification:
  type: "telegram"
  telegram:
    token: "${TELEGRAM_TOKEN}"
    chat_id: "${TELEGRAM_CHAT_ID}"

# 可选：其他通知渠道预留
# notification:
#   type: "wechat"
#   webhook_url: "${WECHAT_WEBHOOK_URL}"
```

## 7. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 股票停牌 | 跳过当日监控，保持上次状态 |
| 数据获取失败 | 重试3次，每次间隔10秒 |
| 新添加的股票 | 首次不发送通知，只记录状态 |
| 节假日（无交易） | 不执行监控，保持上次状态 |
| 状态文件丢失 | 重新初始化，发送通知 |
